При проектировании классов важно знать, что если свойство имеет свои собственные
свойства, например, House имеет окна, двери и др. элементы. Каждый элемент дома
имеет собственные элементы, поэтому присоздании класса Дом, нужно, чтобы каждый
элемент дома был не строкой, а объектом соответствующего класса.

Статические методы - это обычные функции, которые не относятся к объекту. Такие
методы вызываем как правило по имени класса. В таких методах запрещено использовать
self. В классе запрещено создавать функции, вместо них используем статические методы.

Геттеры это методы, которые пишутся для каждого закрытого поля. Они дают доступ
на чтение для этого поля

Сеттеры - это методы для возможности изменения значения свойства объекта

-------------Наследование--------------

При создании потомка класса нужно учитывать наличие конструктора в классе предке.
Если в потомке Вам недостаточно базовых свойств предка, то необходимо добавить
в потомке конструктор класса, где указываем в качестве параметров все параметры
базового конструктора + новые параметры, необходимые текущему объекту. В конструкторе
потомка обязательно в этом случае нужно запустить конструктор базового класса.
