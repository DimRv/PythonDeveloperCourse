Клиент - это ПО, которое делает запросы на сервер. (Браузер это клиент)
Сервер - это ПО, которое ожидает запросы от клиента, обрабатывает их и отправляет ответ клиенту в виде JSON,HTML,XML

Вэб-сервис - это ПО, которое ожидает запрос от клиента и отвечает на этот запрос с помощью JSON

REST API - это механизм взаимодействия клиента и сервера по протоколу HTTP(HTTPS). Взаимодействие между
клиентом и сервером осуществляется только если оба активны. У протокола HTTP есть методы и у каждого
метода HTTP есть назначение:

1) GET - используется для отправки запроса с целью получения данных от сервера. Тело запроса не поддерживается.
Существует возможность отправить на сервер GET параметры. Пример запроса с GET параметрами:
https://server?login=test&role=user В данном запросе есть 2 GET параметра: login и role

2) POST - используется для отправки запроса с телом запроса с целью добавления информации из тела запроса
на сервер. То есть цель этого метода - вставка данных на сервер. Данный метод является неидемпотентным методом.
Если отправить на серверер несколько раз одно и тоже тело запроса, при этом на сервер несколько раз добавится эта
информация. Метод называется идемпотентным если мы отправим несколько раз одинаковое тело запроса и при этом
состояние сервера не изменится, то есть данные не вставляются при каждом запросе. Тело запроса - это обычный
набор данных в виде json или xml...

3) PUT - это метод для отправки в теле запроса нового объекта, который должен заменить существующий объект на сервере.
Если объект на сервере не существует, то он будет добавлен.

4) Если требуется обновлять не весь объект целиком, а лишь какую-то часть объекта, то используйте метод PATCH,
то есть данный метод выполняет частичное обновление объекта

5) DELETE - используется для удаления данных на сервере. Поддерживается тело запроса, но на практике ограничиваются
параметрами
